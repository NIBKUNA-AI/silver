Step 01

Supabase DB 마이그레이션

새로운 Supabase 프로젝트를 생성한 후, SQL Editor에서 다음 파일들의 내용을 순차적으로 실행하십시오.

01. database/schema.sql 전체 테이블 구조 및 기본 RLS 보안 레이어 생성
02. database/admin_settings.sql 로고, 센터명 등 동적 설정 관리 테이블 생성
03. database/create_admin_delete_rpc_v5.sql 관리자 전용 유저 삭제 백엔드 함수(RPC) 생성
04. database/final_fix_signup_trigger.sql 필수 회원가입 시 프로필 자동 생성 엔진
05. init.sql (Root) 기본 프로그램 리스트 및 초기 카테고리 설정

※ 주의: 순서가 틀리면 외래키 참조 에러가 발생할 수 있습니다.

Step 02

스토리지 & 인증 설정

Supabase 웹 대시보드 관리자 메뉴에서 수동으로 설정해야 하는 항목입니다.

메뉴 설정 내용 상태
Storage center-assets, blog-images, attendance-logs 버킷 생성 Public 설정
Auth Providers -> Email -> Confirm Email 스위치 해제 즉시 가입 허용
Step 03

Vercel 환경 변수 세팅

코드 수정 없이 사이트 정보를 변경하는 핵심 단계입니다. Vercel Settings에서 입력하십시오.

환경 변수 (Key) 설명 (Description)
VITE_SUPABASE_URL 새 Supabase 프로젝트 API 주소 (Project URL)
VITE_CENTER_ID 필수 centers 테이블에 등록한 행의 UUID
VITE_SITE_TITLE 브라우저 탭 및 SEO 타이틀 (예: 자라다 OO점)
VITE_CANONICAL_URL 실제 서비스 도메인 (HTTPS 포함 전체 주소)
