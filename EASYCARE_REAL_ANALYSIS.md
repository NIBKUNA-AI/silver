# 🕵️‍♀️ 실제 [이지케어] 시스템 심층 분석 및 내부 구현 타당성 검토

사용자님께서 제공해주신 데모 화면과 영상 자료를 정밀 분석한 결과, 이 시스템은 단순한 센터 관리가 아닌 **"국민건강보험공단 장기요양기관(방문요양, 주야간보호 등) 전용 특화 ERP"**입니다.

보여주신 화면에서 확인된 기능만 수십 가지입니다. 이 중에서 **"우리만의 이지케어"를 만들 때 반드시 필요한 핵심 기능(Essential)**을 솎아내고, **외부 API 연동 없이 내부 로직으로 구현 가능한지** 분석했습니다.

---

## 📊 1. 핵심 기능 분석 및 필수 선정 (Extraction)

> **범례**: 
> 🔴 **API 필수** (외부 연동 없으면 불가능)
> 🟡 **API 권장** (없어도 파일 업로드/수기로 대체 가능 - **내부 구현 가능**)
> 🟢 **내부 구현 100% 가능** (순수 코드와 DB로 제작)

### 1️⃣ 수급자(어르신) 및 직원 관리 (기초 데이터)
화면의 '직원 관리', '수급자 관리' 탭에 해당합니다.

| 실제 기능 | 우리 시스템 구현 가능성 | 필수 여부 | 상세 분석 |
| :--- | :---: | :---: | :--- |
| **수급자 정보 관리** | 🟢 **가능** | **필수** | 등급(1~5등급), 인정번호, 만료일, 낙상/욕창 위험도 기록 관리. |
| **직원(요양보호사) 관리** | 🟢 **가능** | **필수** | 자격증 정보, 배상책임보험 가입 여부, 건강검진 기록 관리. |
| **근로계약서 자동 생성** | 🟢 **가능** | **필수** | 저장된 시급/근로시간 데이터를 바탕으로 계약서 PDF 자동 생성. |
| **문자(SMS) 발송** | 🔴 **불가** | 선택 | 실제 발송은 API 필요. (단, **"발송 내역 기록"** 기능은 내부 구현 가능) |

### 2️⃣ 일정 및 수가 관리 (시스템의 심장)
화면의 '방문일정 관리', 'RFID점검' 탭에 해당합니다.

| 실제 기능 | 우리 시스템 구현 가능성 | 필수 여부 | 상세 분석 |
| :--- | :---: | :---: | :--- |
| **방문 일정표 작성** | 🟢 **가능** | **필수** | 요양보호사와 수급자를 매칭하고 시간(60분, 90분 등)에 따른 자동 스케줄링. |
| **수가(금액) 자동 계산** | 🟢 **가능** | **필수** | **가장 중요.** 공단 고시 수가(평일/야간/휴일/가족요양) 로직을 내장하여 자동 계산. |
| **일정 중복/오류 감지** | 🟢 **가능** | **필수** | 한 요양보호사가 같은 시간에 두 곳에 배정되지 않도록 막는 로직. |
| **RFID 태그 내역 비교** | 🟡 **대체 가능** | **필수** | 정부 서버와 직접 연동은 어려우나, **"공단 엑셀 업로드"** 방식으로 비교 기능 구현 가능. |

| **4대 보험 계산** | 🟢 **가능** | 선택 | 요율만 입력해두면 자동 계산 가능 (실제 신고는 세무사 위임이 많으므로 계산기만 제공). |

| **CMS 자동 출금** | 🔴 **불가** | 선택 | 은행망 연동 필요. (단, **"미납 관리 대장"** 기능으로 수기 관리 대체 가능) |

## 🛠️ 결론: "외부 API 없이 어디까지 되나요?"

**놀랍게도 핵심 기능의 90%는 외부 연동(API) 없이 내부 로직만으로 구현 가능합니다.**

1.  **API 없으면 안 되는 것**:
    *   실제 문자메시지/카톡 발송 (발송 '기록'은 가능)
    *   통장 자동 출금 (CMS)
    *   건강보험공단 서버 실시간 동기화 (RFID 실시간 전송 등)

2.  **내부 로직으로 대체(구현) 가능한 해결책**:
    *   **공단 연동 대체**: 사용자가 공단 사이트에서 엑셀을 받아 업로드하면 -> 데이터를 파싱해서 보여주는 방식 (많은 솔루션이 이렇게 시작함).
    *   **CMS 대체**: 청구서(PDF)만 만들어주고, 입금 여부는 관리자가 "입금완료" 버튼을 누르는 방식.

## ✅ 최종 제안: [EasyCare Lite] 개발 범위 선정

사용자님의 목표인 "우리만의 이지케어"를 구축하기 위해, **API 연동 없이 자체 제작 가능한 필수 기능**만 추렸습니다.

1.  **스마트 스케줄링**: 캘린더 드래그앤드롭으로 요양보호사-수급자 매칭 (수가 자동 계산 포함).
2.  **전자 서류함**: 상담일지, 욕구사정, 계약서를 PDF로 출력할 필요 없이 웹에서 작성/서명/저장.
3.  **급여/청구 계산기**: 복잡한 야간/휴일 가산 수당을 자동으로 계산해주는 엔진.
4.  **평가 신호등(대시보드)**: "김철수 어르신, 3월 상담일지가 누락되었습니다!"라고 빨간 불 켜주는 기능.
